# DeltaXY brand style guide

*Version 3.0 — March 31, 2026*
*Calm, light, long-term. Inspired by Zen principles of restraint, space, and composure.*

---

## Brand direction

DeltaXY should feel like **relief**.

Not hype. Not startup energy. Not glossy futurism.

The visual and verbal system should create a simple reaction in the buyer:

- this feels calm
- this feels under control
- this feels trustworthy
- this company will still be here in 10 years

We borrow from Zen-influenced design in the right way:
- meaningful empty space
- muted natural tones
- quiet hierarchy
- asymmetry with balance
- restraint over decoration
- long-term clarity over trend-chasing

We do **not** borrow the costume:
- no brush-script gimmicks
- no rising-sun motifs
- no bamboo/spa clichés
- no ornamental “Japanese” aesthetics pasted onto enterprise software

## Design principles

1. **Professional first.** We build for regulated teams, not trend-driven startup audiences.
2. **Light mode default.** The interface should feel open, breathable, and office-friendly.
3. **Warm, not cold.** Surfaces should feel closer to paper, stone, and daylight than chrome or glass.
4. **Restraint wins.** Every color, line, and component must earn its place.
5. **Long-term over trendy.** If it feels like a 2026 AI landing page pattern, remove it.
6. **Whitespace is functional.** Space reduces pressure and improves comprehension.
7. **Proof over performance.** Metrics, workflow evidence, and clarity matter more than visual spectacle.
8. **No blue-led identity.** DeltaXY should not look like another blue SaaS company.
9. **No gradients, neon, or glow.** Calm systems don’t shout.
10. **Readable always beats clever.** Especially in dense, review-heavy enterprise workflows.

## Voice and tone

### Brand voice
DeltaXY speaks like a calm expert in a high-stakes room.

We are:
- reassuring
- precise
- measured
- clear
- grounded

We are not:
- flashy
- aggressive
- overfamiliar
- mystical
- “AI revolutionary”

### What it should sound like
- “Document workflows that feel under control.”
- “Built for sensitive, review-heavy work.”
- “Reviewable outputs. Measurable accuracy. Production delivery in weeks.”

### What it should not sound like
- “Revolutionize your operations with next-gen AI.”
- “Supercharge your team with autonomous agents.”
- “Cutting-edge transformation for modern enterprises.”

### Copy rules
- Sentence case always
- Short paragraphs
- Fewer claims, stronger proof
- Calm verbs over hype verbs
- Benefits framed as relief and confidence, not excitement

## Typography

Typography should feel calm, literary, and exact.

Not futuristic. Not corporate-default. Not ornamental.

### Font pairing

#### Display / headings
**Zen Old Mincho**

Use for:
- H1
- H2
- selected editorial pull quotes
- occasional section titles

Why:
- calm authority
- timeless feel
- slight literary weight
- introduces long-term character without theatrics

#### UI / body / navigation
**Zen Kaku Gothic New**

Use for:
- body text
- navigation
- buttons
- labels
- forms
- tables
- captions

Why:
- clean and modern
- warm without being soft
- readable in dense interfaces
- keeps the system grounded and usable

#### Data / numeric content
**Zen Kaku Gothic New** with `tabular-nums`

Use for:
- metrics
- dates
- extraction values
- tables
- IDs
- accuracy numbers

We do **not** force a separate monospace family unless readability absolutely demands it.
The brand should feel unified and calm.

### Type scale

| Style | Font | Weight | Size / Line height | Use |
|-------|------|--------|-------------------|-----|
| **H1** | Zen Old Mincho | 500 | 44px / 56px | Hero headlines |
| **H2** | Zen Old Mincho | 500 | 32px / 42px | Section headers |
| **H3** | Zen Kaku Gothic New | 500 | 22px / 32px | Card and subsection titles |
| **H4** | Zen Kaku Gothic New | 500 | 17px / 26px | Labels, small headings |
| **Body** | Zen Kaku Gothic New | 400 | 16px / 28px | Main copy |
| **Body small** | Zen Kaku Gothic New | 400 | 14px / 24px | Secondary text |
| **Caption** | Zen Kaku Gothic New | 400 | 12px / 18px | Metadata, timestamps |
| **Numeric / data** | Zen Kaku Gothic New | 500 | 14px / 22px | Tables, metrics, values |

### Typography rules

1. Sentence case only
2. No all caps labels
3. No heavy bold beyond what is necessary
4. H1 and H2 should feel spacious, never compressed
5. Letter spacing stays natural — no dramatic tracking
6. Numeric content uses tabular numbers for alignment
7. Body copy should feel breathable, not dense

## Component patterns

### Buttons

| Type | Background | Text | Border | Use |
|------|------------|------|--------|-----|
| **Primary** | Ink Sage `#4E5B4D` | Rice `#FCFAF7` | None | Main actions |
| **Secondary** | Rice `#FCFAF7` | Ink Sage `#4E5B4D` | 1px Ink Sage 40 | Secondary actions |
| **Tertiary** | Transparent | Ink 700 `#59534D` | None | Low-emphasis actions |
| **Danger** | Oxide `#9B4E43` | Rice `#FCFAF7` | None | Destructive actions |

Button specs:
- Height: 38px default
- Padding: 12px 20px
- Border radius: 6px
- Font: Zen Kaku Gothic New Medium, 14px
- No all caps
- No shiny fills, shadows, or gradients

### Cards

- Background: Rice `#FCFAF7`
- Border: 1px Stone 200 `#E2DBD1`
- Border radius: 8px
- Padding: 20px
- Shadow: `0 1px 2px rgba(35,33,29,0.04)`
- Cards should feel like well-made paper panels, not floating glass objects

### Tables

Tables remain a core DeltaXY pattern.

| Element | Style |
|---------|-------|
| Header row | Paper 100 `#F2EDE6`, Ink 900 text, Zen Kaku Gothic New Medium 13px |
| Body rows | Rice `#FCFAF7`, alternating rows can use `#F8F4EE` lightly |
| Row hover | Ink Sage 20 `#E7ECE4` |
| Selected row | Ink Sage 20 with 3px left border in Ink Sage 80 |
| Cell text | Zen Kaku Gothic New 14px, Ink 800 |
| Numeric data | Zen Kaku Gothic New 14px, tabular nums, right-aligned |

### Status badges

| Status | Background | Text | Border |

## Layout

### Composition principles

1. **Use space deliberately.** Let sections breathe.
2. **Avoid crowded hero layouts.** One core message. One strong CTA.
3. **Prefer asymmetry with balance.** Break rigid template symmetry where useful.
4. **Do not rely on card grids everywhere.** Use editorial rhythm.
5. **Let proof have room.** Metrics and case studies should feel calm, not salesy.

### Spacing

| Token | Value | Use |
|-------|-------|-----|
| `xs` | 4px | Tight icon/text relationships |
| `sm` | 8px | Related controls |
| `md` | 16px | Field spacing, list spacing |
| `lg` | 24px | Card spacing |
| `xl` | 40px | Section spacing |
| `2xl` | 64px | Major page breaks |
| `3xl` | 96px | Hero and major editorial spacing |

### Border radius

| Token | Value | Use |
|-------|-------|-----|
| `sm` | 4px | Badges, small controls |
| `md` | 6px | Buttons, inputs |
| `lg` | 8px | Cards, modals |

No pills. No oversized radius. No soft consumer-app curves.

---

## Motion

Motion should reinforce calm.

### Rules
- soft fade or slight upward settle only
- 160–220ms for microinteractions
- 220–320ms for section transitions
- no bounce
- no overshoot
- no scrolljacking
- no attention-seeking animations

If motion does not improve clarity, remove it.

---

## Imagery and texture

### Good imagery direction
- daylight
- paper, desks, documents, architecture, surfaces
- quiet work environments
- natural materials
- close-up detail rather than generic office theatre

### Avoid
- stock-photo teams pointing at screens
- abstract AI blobs
- glowing dashboards floating in space
- overtly “Japanese” decorative motifs
- spiritual/wellness imagery

### Texture
Use extremely subtle texture where helpful:
- paper grain
- soft wash
- natural surface depth

Texture must remain nearly invisible.
It supports calm; it should never become the design.

---

## Website structure guidance

The homepage should feel like a measured sales argument:

1. relief
2. what we do
3. proof
4. how it works
5. partner trust
6. call to action

Do not start with feature grids.
Do not start with platform language.
Do not start with “AI transformation.”

### Hero guidance
The hero should feel like a deep breath.

Use:
- one calm headline
- one supporting paragraph
- one primary CTA
- one proof strip

Avoid:
- multiple CTA buttons fighting for attention
- dashboard-collage hero art
- loud gradient backgrounds
- too many metrics above the fold

---

## Product UI guidance

Zen influence applies to the **feel**, not to the density of the workflow.

Product UI must still be:
- explicit
- table-friendly
- review-friendly
- operationally clear

That means:
- warm light surfaces
- disciplined typography
- low-noise controls
- clear states
- visible structure

Not:
- ultra-empty minimalism that hides necessary information
